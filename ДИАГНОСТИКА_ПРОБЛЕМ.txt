ДИАГНОСТИКА ПРОБЛЕМ С САЙТОМ
=============================

ПРОБЛЕМА: Сервер запускается, но сайт не открывается

ВОЗМОЖНЫЕ ПРИЧИНЫ:
==================

1. БРАУЗЕР НЕ ОТКРЫВАЕТСЯ АВТОМАТИЧЕСКИ
   - Откройте браузер вручную
   - Перейдите по адресу: http://127.0.0.1:8000/
   - Или попробуйте: http://localhost:8000/

2. ПОРТ 8000 ЗАНЯТ ДРУГИМ ПРОЦЕССОМ
   - Закройте все программы
   - Перезагрузите компьютер
   - Запустите сервер на другом порту

3. ФАЙРВОЛ БЛОКИРУЕТ СОЕДИНЕНИЕ
   - Отключите файрвол временно
   - Добавьте исключение для Python

4. АНТИВИРУС БЛОКИРУЕТ СОЕДИНЕНИЕ
   - Отключите антивирус временно
   - Добавьте исключение для папки проекта

5. ПРОБЛЕМЫ С DJANGO
   - Проверьте, что миграции применены
   - Проверьте, что нет ошибок в коде

РЕШЕНИЯ:
=========

РЕШЕНИЕ 1 (Проверка браузера):
1. Откройте браузер вручную
2. Введите адрес: http://127.0.0.1:8000/
3. Если не работает, попробуйте: http://localhost:8000/

РЕШЕНИЕ 2 (Другой порт):
1. Запустите сервер на порту 8080:
   python manage.py runserver 127.0.0.1:8080
2. Откройте: http://127.0.0.1:8080/

РЕШЕНИЕ 3 (Проверка процессов):
1. Откройте диспетчер задач
2. Найдите процессы python.exe
3. Завершите их все
4. Запустите сервер заново

РЕШЕНИЕ 4 (Проверка файрвола):
1. Откройте настройки файрвола
2. Добавьте исключение для Python
3. Или отключите файрвол временно

РЕШЕНИЕ 5 (Полная перезагрузка):
1. Закройте все программы
2. Перезагрузите компьютер
3. Запустите ПРОСТОЙ_ЗАПУСК.bat

ПРОВЕРКА РАБОТЫ СЕРВЕРА:
========================

1. Запустите сервер
2. В командной строке должно появиться:
   "Starting development server at http://127.0.0.1:8000/"
3. Если есть ошибки - исправьте их
4. Если нет ошибок - откройте браузер вручную

АЛЬТЕРНАТИВНЫЕ АДРЕСА:
======================

Попробуйте эти адреса:
- http://127.0.0.1:8000/
- http://localhost:8000/
- http://0.0.0.0:8000/
- http://192.168.1.1:8000/ (ваш IP)

ПРОВЕРКА СЕТИ:
==============

1. Откройте командную строку
2. Выполните: ping 127.0.0.1
3. Если пинг проходит - сеть работает
4. Если не проходит - проблема с сетью

ПРОВЕРКА ПОРТА:
===============

1. Откройте командную строку
2. Выполните: netstat -an | findstr 8000
3. Если порт занят - используйте другой
4. Если порт свободен - проблема в другом
